**Node `nodeType` values:**
| nodeType | Description |

**Edge `relType` values:**
| relType | Description |
|---|---|
| `CHAIN_MEMBER` | Stock belongs to SupplyChain (structural, skipped in peer list) |

| `SUPPLY_CHAIN` | Upstream/downstream supply relationship |
| `FINANCIAL_RELATION` | Banking, insurance, or credit relationship |
| `EQUITY_HOLDING` | Ownership stake |
| `MACRO_FACTOR` | Stock is sensitive to a MacroFactor node |
| `FEEDS_INTO` | Generic directed flow (skipped in peer list) |
| `ROOT_CAT` / `CAT_MACRO` / `CAT_CHAIN` / `ROOT_MACRO` / `MACRO_CHAIN` | Structural edges (all skipped in peer list) |

**Structural edges excluded from peer display (SKIP_REL):**
`CHAIN_MEMBER`, `FEEDS_INTO`, `MACRO_CHAIN`, `ROOT_CAT`, `CAT_MACRO`, `CAT_CHAIN`, `ROOT_MACRO`

### `GET /health`

Polled every 3 seconds until `ready: true`. Frontend stops polling once ready.

**Response:**
```json
{
  "ready":         true,
  "startup_error": null
}
```

- `ready`: `false` while startup agents are running
- `startup_error`: string with agent errors, or `null` on clean start

**Startup calculation chain** (`backend/startup.py` ŌåÆ `run_startup()`):

```
Step 1 (sequential):
  global_macro.run_global_macro_analysis()
  ŌööŌöĆ Fetches 8 cross-asset instruments via yfinance:
       ^TNX/TIP   ŌåÆ analyse_real_yield()   ŌĆö 20% weight
       DX-Y.NYB   ŌåÆ analyse_dxy()          ŌĆö 20% weight
       USDTHB=X   ŌåÆ analyse_thb()          ŌĆö 5%  weight
       CL=F/BZ=F  ŌåÆ analyse_crude_oil()    ŌĆö 12% weight
       HG=F       ŌåÆ analyse_copper()       ŌĆö 15% weight
       GC=F       ŌåÆ analyse_gold()         ŌĆö 3%  weight
       EEM/THD    ŌåÆ analyse_em_flows()     ŌĆö 15% weight
       MCHI/KWEB  ŌåÆ analyse_china_pulse()  ŌĆö 10% weight
  ŌööŌöĆ Composite = weighted_avg(8 risk_scores), clamped [0,100]
  ŌööŌöĆ Derives cycle_quadrant, sector_adjustments, macro_bias_summary

Step 2 (concurrent, both depend on Step 1 macro output):
  market_risk.run_fragility_monitor()     ŌåÆ startup_state["market_risk"]
  sector_screener.run_sector_screener()   ŌåÆ startup_state["sector_screener"]

  startup_state["ready"] = True   ŌåÉ frontend polling stops here
```

**`macro` ŌĆö How each sub-signal is computed** (`src/agents/global_macro.py`):

Each of the 8 signal functions shares the same structure: fetch `history("Close")`, then compute:
- `momentum(window)` = `(price[-1] / price[-window] - 1) * 100`
- `zscore(window=60)` = `(price[-1] - mean(last 60)) / std(last 60)`
- Rule-based `risk_score` (0ŌĆō100) and `signal` string based on momentum + z-score thresholds

| Signal | Source ticker(s) | Key logic |
|---|---|---|
| `real_yield` | `^TNX`, `TIP` | Yield level > 4.5% + TIP falling ŌåÆ HIGHLY_RESTRICTIVE_RISING (score 85) |
| `dxy` | `DX-Y.NYB` | z-score > 1.5 and mom_20 > 1.5% ŌåÆ STRONG_USD_HEADWIND (score 80) |
| `thb` | `USDTHB=X` | z-score > 1.5 and mom_20 > 1% ŌåÆ THB_SHARP_DEPRECIATION (score 75) |
| `crude_oil` | `CL=F` | price > 90 and mom_20 > 3% ŌåÆ STAGFLATION_RISK (score 70) |
| `copper` | `HG=F` | z-score < -1.0 and mom_60 < -5% ŌåÆ RECESSION_WARNING (score 80) |
| `gold` | `GC=F` | z-score > 1.5 + tnx falling ŌåÆ STRONG_RISK_OFF (score 75) |
| `em_flows` | `EEM`, `SPY`, `THD` | `em_alpha = EEM_mom20 - SPY_mom20`; < -3% and THD < -1% ŌåÆ EM_OUTFLOW_PRESSURE (score 80) |
| `china` | `MCHI`, `KWEB` | MCHI_mom20 > 3% and KWEB_mom20 > 2% ŌåÆ CHINA_ACCELERATING (score 15) |

**Composite macro formula:**
```
composite_macro_risk = ╬Ż(weight[signal] ├Ś risk_score[signal])
  weights: real_yield=0.20, dxy=0.20, em_flows=0.15, copper=0.15,
           oil=0.12, china=0.10, thb=0.05, gold=0.03
```

**Cycle quadrant derivation:**
```
growth_proxy  = mean([50 + copper.z*12, 100-em_flows.risk, 100-china.risk])
inflation_proxy = mean([oil.risk, real_yield.risk])

growth_rising=True  + inflation_rising=False ŌåÆ GOLDILOCKS
growth_rising=True  + inflation_rising=True  ŌåÆ OVERHEAT
growth_rising=False + inflation_rising=True  ŌåÆ STAGFLATION
growth_rising=False + inflation_rising=False ŌåÆ RECESSION_RISK
```

**Sector adjustments** ŌĆö event flags derived from signal values, applied per sector via lookup table:
```python
# Example for TRANSPORT sector:
{"oil_up": -8, "oil_down": +8, "gold_up": -1, "dxy_up": -2, "copper_up": +3,
 "real_yield_rising": -3, "china_up": +4, "em_inflow": +2, "thb_weakening": +2}
# Active flags summed, clamped [-20, +20]
```

**`market_risk` ŌĆö How it's computed** (`src/agents/market_risk.py` ŌåÆ `run_fragility_monitor()`):

Three-layer framework. Each indicator produces a `risk_score` [0ŌĆō100]:

**Layer 1 ŌĆö Regime (weight 40%)**
| Indicator | Function | How |
|---|---|---|
| `spx_200dma` | `calculate_spx_200dma_buffer()` | `(SPX - SMA200) / SMA200 * 100`; > 5% ŌåÆ score 15, < -5% ŌåÆ score 90 |
| `yield_curve` | `calculate_yield_curve_spread()` | `(^TNX - ^IRX) * 100` bps; > 50bp ŌåÆ score 20, inverted ŌåÆ score 85 |
| `hy_credit_spread` | `calculate_hy_credit_spread()` | HYG yield ŌłÆ 10Y risk-free = OAS bps; < 300bp ŌåÆ score 15, > 600bp ŌåÆ score 95 |

**Layer 2 ŌĆö Fragility (weight 40%)**
| Indicator | Function | How |
|---|---|---|
| `breadth_50dma` | `calculate_breadth_above_50dma()` | % of SET100 stocks above 50DMA; Ōēź 70% ŌåÆ score 15, < 40% ŌåÆ score 85 |
| `rsp_spy_ratio` | `calculate_rsp_spy_ratio()` | RSP/SPY z-score vs 1Y mean; > 0.5Žā ŌåÆ score 15, < -1Žā ŌåÆ score 80 |

**Layer 3 ŌĆö Trigger (weight 20%)**
| Indicator | Function | How |
|---|---|---|
| `vix_spot` | `calculate_vix_level()` | VIX < 15 ŌåÆ score 10, VIX > 35 ŌåÆ score 95 |
| `vix_term_structure` | `calculate_vix_term_structure()` | VIX3M ŌłÆ VIX9D roll yield; > 3 contango ŌåÆ score 10, < -3 backwardation ŌåÆ score 90 |

**Composite formula:**
```
composite_risk = (0.40 ├Ś mean(regime_scores))
              + (0.40 ├Ś mean(fragility_scores))
              + (0.20 ├Ś mean(trigger_scores))

regime_label: < 25 ŌåÆ LOW_RISK, < 45 ŌåÆ MODERATE, < 65 ŌåÆ ELEVATED,
              < 80 ŌåÆ HIGH, Ōēź 80 ŌåÆ CRITICAL

confidence = 100 - (std_dev(all_scores) / 50 * 100)
```

**`sector_screener` ŌĆö How it's computed** (`src/agents/sector_screener.py` ŌåÆ `run_sector_screener()`):

Scores each sector ETF across 4 dimensions vs `^SET.BK` benchmark:

```
score = 50 (neutral anchor)
      + clip(mom_20d * 1.5, -20, +20)          # momentum
      + clip(rs_vs_index * 2.0, -20, +20)       # relative strength = sector_mom20 - bench_mom20
      + (breadth_pct - 50) * 0.30               # internal breadth above 50DMA
      + clip(volume_flow * 8, -10, +10)          # OBV-style flow signal
      + macro_adjustment                          # from global_macro sector_adjustments

signal:  Ōēź 70 ŌåÆ STRONG_BUY_SECTOR, Ōēź 58 ŌåÆ MILD_OVERWEIGHT,
         Ōēź 45 ŌåÆ NEUTRAL_HOLD, Ōēź 35 ŌåÆ MILD_UNDERWEIGHT, < 35 ŌåÆ AVOID_SECTOR
```

Sector rotation regime detected from top 3 gate-passing sectors:
```
TECH + CONSUMER_DISC in top 3  ŌåÆ EARLY_CYCLE_GROWTH
ENERGY + MATERIALS in top 3    ŌåÆ LATE_CYCLE_INFLATIONARY
UTILITIES + HEALTH in top 3    ŌåÆ DEFENSIVE_LATE_BEAR
FINANCIALS in top 3            ŌåÆ MID_CYCLE_EXPANSION
else                           ŌåÆ TRANSITION_MIXED
```

**Macro signal enums:**
- `macro_regime`: `"RISK_ON_EXPANSION"` | `"NEUTRAL_GROWTH"` | `"CAUTIOUS_ELEVATED"` | `"RISK_OFF_DEFENSIVE"` | `"CRISIS_PRESERVE_CAPITAL"`
- `cycle_quadrant`: `"GOLDILOCKS"` | `"OVERHEAT"` | `"STAGFLATION"` | `"RECESSION_RISK"`
- Per-signal `signal` strings: e.g. `"RESTRICTIVE_RISING"`, `"MILD_USD_STRENGTH"`, `"THB_STABLE"`, `"MODERATE_MOVE"`, `"GROWTH_POSITIVE"`, `"MILD_RISK_OFF"`, `"EM_UNDERPERFORMING"`, `"CHINA_RECOVERY"`
- Per-signal `macro_bias` strings: `"CAUTIOUS"`, `"NEUTRAL"`, `"CAUTIOUS_EM"`, `"DEFENSIVE"`, `"MILD_CYCLICAL"`, etc.

### `GET /prices?tickers=AAV.BK,BDMS.BK,^SET.BK,CL=F,GC=F`

Fetched once after `/health` is ready. Called with all `RANKED_TICKERS` (SET100 `.BK` tickers) plus macro symbols `^SET.BK`, `CL=F`, `GC=F`.

**Query params:**
- `tickers`: comma-separated list of yfinance ticker symbols

**Response:**
```json
{
  "AAV.BK":  { "price": 3.20, "change_pct": 3.23 },
  "BDMS.BK": { "price": 28.5, "change_pct": -0.70 },
  "^SET.BK": { "price": 1457.3, "change_pct": 0.50 },
  "CL=F":    { "price": 82.4, "change_pct": 1.54 },
  "GC=F":    { "price": 2340.5, "change_pct": 0.88 }
}
```

- Keys use the exact ticker string passed in (including `.BK`, `=F`, `^` prefixes)
- A ticker that fails returns `null` as value: `"BAD.BK": null`

**How it's computed** (`server.py` ŌåÆ `_fetch()` inside `/prices`):

```python
# Per ticker ŌĆö serial loop (known bottleneck, see AUDIT_REPORT P3)
fi    = yf.Ticker(t).fast_info          # lightweight yfinance endpoint
price = fi.last_price
prev  = fi.previous_close
chg   = (price - prev) / prev * 100    # % change from previous close
result[t] = { "price": round(price, 2), "change_pct": round(chg, 2) }
```

`fast_info` is a lightweight yfinance call (no full `.info` fetch). Returns `last_price` and `previous_close` from the exchange's last session.

### `GET /correlations?tickers=AAV.BK,BDMS.BK,TLT,UUP,MCHI,GC=F,CL=F,^SET.BK&period=30d&interval=1d`

Triggered by `selectStock()` ŌĆö called whenever a stock is selected.

**Tickers sent:** ego stock's direct neighbours (non-structural edges) + always includes these 6 macro reference tickers:
```
TLT, UUP, MCHI, GC=F, CL=F, ^SET.BK
```

**Query params:**
- `tickers`: comma-separated list
- `period`: default `"30d"`
- `interval`: default `"1d"`

**Response:** symmetric N├ŚN Pearson Žü matrix (only non-zero entries, no self-pairs):
```json
{
  "AAV.BK": {
    "BDMS.BK": 0.4521,
    "TLT":     -0.3210,
    "CL=F":    0.6843,
    "^SET.BK": 0.7120
  },
  "BDMS.BK": {
    "AAV.BK":  0.4521,
    "^SET.BK": 0.6800
  }
}
```

- Keys are exact yfinance ticker strings
- `MacroRegimePanel` looks up correlations using both `TICKER` and `TICKER.BK` variants
- Lookup tries: `correlations[ticker][sym]` ŌåÆ `correlations[ticker][sym.replace('.BK','')]` ŌåÆ `correlations[ticker][sym+'.BK']`

**How it's computed** (`server.py` ŌåÆ `_compute()` inside `/correlations`):

```python
# 1. Batch-download all tickers in one call
raw  = yf.download(ticker_list, period=period, interval=interval, auto_adjust=True)
closes  = raw["Close"]                        # MultiIndex DataFrame
returns = closes.pct_change().dropna(how="all")

# 2. Per-pair Pearson Žü  (current implementation ŌĆö O(N┬▓) pairs)
for col in returns.columns:
    for other in returns.columns:
        if col == other: continue
        paired = pd.concat([returns[col].dropna(),
                            returns[other].dropna()], axis=1).dropna()
        if len(paired) < 5: continue
        rho = float(np.corrcoef(paired.iloc[:,0], paired.iloc[:,1])[0,1])
        result[col][other] = round(rho, 4)
```

`period="30d"` gives ~22 trading-day observations per pair (minimum 5 required to emit a value).

**Full calculation chain** (`backend/analysis.py` ŌåÆ `run_analysis(ticker)`):

**Step 1 ŌĆö Data fetch** (`src/data/providers.py` ŌåÆ `fetch_all_data(ticker)`):
```python
stock   = yf.Ticker(ticker)
history = stock.history(period="3y")          # 3 years OHLCV

# Enrichment for non-US tickers (.BK, .HK, etc.):
```

**Step 2 ŌĆö Fundamental analysis** (`src/agents/fundamental_agent.py` + `src/agents/calculator.py`):

All metrics read from yfinance financial statements (`stock.financials`, `stock.balance_sheet`, `stock.cashflow`). Row labels matched with case-insensitive normalisation to handle `.BK` ticker variations.

| Field | Function | Formula |
|---|---|---|
| `roic` | `calculate_roic()` | `NOPAT / Invested_Capital` where `NOPAT = EBIT ├Ś (1 ŌłÆ tax_rate)`, `IC = (Debt + Equity) ŌłÆ Cash` |
| `wacc` | `calculate_wacc()` | `(E/V ├Ś Re) + (D/V ├Ś Rd ├Ś (1ŌłÆTc))`, CAPM: `Re = Rf(4.3%) + ╬▓ ├Ś ERP(5.0%)` |
| `roic_wacc_spread` | computed | `roic ŌłÆ wacc` |
| `moat` | computed | spread > 10% ŌåÆ WIDE, > 5% ŌåÆ NARROW, > 0% ŌåÆ MARGINAL, Ōēż 0 ŌåÆ NONE |
| `sloan_ratio` | `calculate_sloan_ratio()` | `(Net Income ŌłÆ CFO) / Total Assets` ŌĆö > 0.2 flags earnings inflation |
| `fcf_quality` | `calculate_fcf_quality()` | `(CFO ŌłÆ CapEx) / Net Income` ŌĆö > 1.0 = exceptional, < 0.3 = red flag |
| `altman_z` | `calculate_altman_z()` | `1.2├ŚX1 + 1.4├ŚX2 + 3.3├ŚX3 + 0.6├ŚX4 + 0.99├ŚX5` (full 5-variable public model) |
| `altman_zone` | computed | Z > 2.99 ŌåÆ SAFE, 1.81ŌĆō2.99 ŌåÆ GREY, < 1.81 ŌåÆ DISTRESS |
| `asset_turnover` | `calculate_asset_turnover()` | `Revenue / Total Assets` |
| `cash_conversion_cycle` | `calculate_ccc()` | `DIO + DSO ŌłÆ DPO` (days inventory + receivables ŌłÆ payables) |
| `sortino` | `calculate_rolling_sortino()` | `(mean_excess_return ├Ś 252) / downside_deviation` where `excess = returns ŌłÆ Rf/252` |
| `beta` | from `ticker_info` | yfinance local beta (vs SET50 for .BK tickers) |

**Alpha score** (`generate_alpha_score()`) ŌĆö 5-dimension composite [0ŌĆō100]:

```
Dimension 1 ŌĆö Economic Value Add  (max 30 pts)
  ROICŌłÆWACC spread:  > 10% ŌåÆ +30,  > 5% ŌåÆ +22,  > 0% ŌåÆ +12,  Ōēż 0 ŌåÆ +0

Dimension 2 ŌĆö Earnings Quality    (max 20 pts)
  Sloan ratio:  |sloan| < 0.05 ŌåÆ +20,  < 0.10 ŌåÆ +14,  < 0.20 ŌåÆ +6,  Ōēź 0.20 ŌåÆ +0

Dimension 3 ŌĆö FCF Quality         (max 10 pts)  [NEW vs original]
  FCF/NI:  > 1.0 ŌåÆ +10,  > 0.6 ŌåÆ +7,  > 0.3 ŌåÆ +3,  Ōēż 0.3 ŌåÆ +0

Dimension 4 ŌĆö Survival            (max 20 pts)
  Altman Z:  > 2.99 ŌåÆ +20,  > 1.81 ŌåÆ +10,  Ōēż 1.81 ŌåÆ +0

Dimension 5 ŌĆö Risk-Adj Return     (max 20 pts)
  Sortino:  > 2.5 ŌåÆ +20,  > 1.5 ŌåÆ +15,  > 1.0 ŌåÆ +9,  > 0.5 ŌåÆ +4

Beta penalty (regime-scaled):
  regime_multiplier = 1.0 + max(0, (composite_risk ŌłÆ 40) / 120)  ŌåÆ [1.0, 1.5├Ś]
  beta > 2.0 ŌåÆ ŌłÆ15├Śmult,  beta > 1.5 ŌåÆ ŌłÆ10├Śmult,  beta > 1.3 ŌåÆ ŌłÆ5├Śmult

Macro sector adjustment: ┬▒0..20 from global_macro.sector_adjustments[sector]

Final score = clamp(raw_score + sector_adj, 0, 100)
```

**Signal thresholds:**
```
alpha Ōēź 75 ŌåÆ STRONG_BUY
alpha Ōēź 55 ŌåÆ BUY
alpha Ōēź 40 ŌåÆ NEUTRAL
alpha Ōēź 25 ŌåÆ SELL
alpha <  25 ŌåÆ STRONG_SELL
```

**Step 3 ŌĆö Technical analysis** (`src/agents/technical.py` ŌåÆ `run_technical_analysis()`):

Data: `yf.Ticker(ticker).history(period="1y", interval="1d")` (minimum 60 bars required).

**Stage A ŌĆö Price regime detection** (`detect_price_regime()`):
```
ADX(14)  = Average Directional Index
BB_width = (upper_BB ŌłÆ lower_BB) / middle_BB ├Ś 100

BREAKOUT:  price > yesterday's upper_BB  AND  BB_width < 4.0%
TRENDING:  ADX > 25
RANGING:   ADX Ōēż 25  (default)
```

**Stage B ŌĆö Strategy selection** (`optimise_strategy()`):
All three strategies are back-simulated on the last 60 days. A rolling Sharpe (annualised, Žā of next-bar returns) is computed per strategy. Regime fit bonus added:

| Regime | MOMENTUM bonus | MEAN_REVERSION bonus | BREAKOUT bonus |
|---|---|---|---|
| TRENDING | +0.5 | ŌłÆ0.3 | 0.0 |
| RANGING | ŌłÆ0.2 | +0.5 | ŌłÆ0.1 |
| BREAKOUT | +0.1 | ŌłÆ0.2 | +0.5 |

**Stage C ŌĆö Execute winning strategy:**

| Strategy | Entry | TP | SL | Signal conditions |
|---|---|---|---|---|
| `MOMENTUM` | current price | `entry + 3├ŚATR14` (snapped to resistance if closer) | `entry ŌłÆ 2├ŚATR14` | MACD histogram crosses positive (+35) + RSI 50ŌĆō75 (+30) + price > MA20 (+20) + MACD > 0 (+15) |
| `MEAN_REVERSION` | current price | Middle Bollinger Band | `entry ŌłÆ 1.5├ŚATR14` | RSI < 30 (+40) or RSI < 40 (+25) + %B < 0.05 (+30) + RSI divergence (+20) |
| `BREAKOUT` | current price | `entry + 4├ŚATR14` | `entry ŌłÆ 1.5├ŚATR14` | Price > 20d high (+35) + above upper BB (+25) + volume > 1.3├Ś avg (+25) + prior BB compression (+15) |

R:R gate: if `rr_ratio < 1.5`, `signal_strength -= 20` and warning note appended.

**Error responses:**
- `503` ŌĆö server still initialising (`startup_state.ready == false`)
- `404` ŌĆö ticker not found or yfinance returned no data
